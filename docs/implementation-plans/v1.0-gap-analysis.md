# v1.0 Feature Gap Analysis: playwright-rust vs playwright-python

**Last Updated:** 2026-05-16

**Baseline:** playwright-rust v0.12.0 vs playwright-python (Playwright v1.60+)

## Goal

Comprehensive API gap analysis comparing this repo's Rust bindings against the official Playwright Python bindings. Organized by priority to guide v1.0 development.

---

## Coverage Summary

Full Python API parity was achieved in v0.12.0. v0.13.0 (code complete
on `main`, awaiting tag cut) closes the remaining Playwright 1.59 gap
‚Äî `Page::screencast`, `Debugger`, `Page::pick_locator`, CDP event side
on `CDPSession`, `AriaSnapshotOptions`, `Tracing.start({live: true})`,
`Page::aria_snapshot`, `Locator::normalize`, `LaunchOptions::artifacts_dir`
‚Äî and adds two companion crates (`playwright-rs-macros` for the
compile-time-validated `locator!()` macro, `playwright-rs-trace` for
programmatic trace-file consumption). The bundled driver was bumped
from 1.59.1 ‚Üí 1.60.0 on `main` on 2026-05-16; the bump is source-compatible
(none of the upstream-removed 1.60 APIs were exposed in this crate),
but the new 1.60 surface (HAR Tracing API, `Locator::drop`, expanded
BrowserContext lifecycle events, etc.) is tracked separately under
"v0.14.0 ‚Äî Playwright 1.60 Completeness" below for incremental pre-v1.0
rollout. With v0.13.0 staged, playwright-rust matches the Playwright
1.59 surface and is slightly ahead of playwright-python.

v1.0.0 follows after multi-month dogfooding with real-world projects ‚Äî
API polish, ergonomics refinements, and performance tuning rather than
new surface area.

- `Page::screenshot` / `Locator::screenshot` were treated as complete, but

  landing site, where an animated tab indicator raced an element screenshot
  (the idiomatic fix is `animations: "disabled"`). Tracked and addressed under
  v0.14.0. Same lesson: grep the source, not this doc.

## P0 ‚Äî Critical for v1.0

### Locator: `get_by_*` Methods

These are *the* modern Playwright locator pattern. Without them, users cannot write idiomatic Playwright tests.

- [x] `get_by_role(role)` ‚Äî Locate by ARIA role (most important)
- [x] `get_by_text(text)` ‚Äî Locate by text content
- [x] `get_by_label(text)` ‚Äî Locate by associated label
- [x] `get_by_placeholder(text)` ‚Äî Locate by placeholder text
- [x] `get_by_test_id(test_id)` ‚Äî Locate by test ID attribute
- [x] `get_by_alt_text(text)` ‚Äî Locate by alt text
- [x] `get_by_title(text)` ‚Äî Locate by title attribute

These must also be available on **Page** and **Frame** (they delegate to the main frame's locator).

### v0.14.0 ‚Äî "Playwright 1.60 Completeness"

üÜï **Planned.** Tracks the new API surface introduced upstream in
Playwright 1.60 (released 2026-05-11). Driver itself bumped on
`main` 2026-05-16 ‚Äî these items add Rust bindings for the new
methods/options. Priority labels match the P0/P1/P2/P3 buckets above.

**P1 ‚Äî High value:**

- [ ] `Locator::drop(options)` ‚Äî simulate external drag-and-drop of files or clipboard data onto an element. Dispatches `dragenter`/`dragover`/`drop` with a synthetic `DataTransfer`. Distinct from existing `drag_to(target)` (intra-page drag); this is the canonical primitive for upload-zone tests. Options: `files: Vec<FilePayload>` and/or `data: HashMap<String, String>` (MIME-typed string payloads).
- [ ] `BrowserContext` lifecycle event mirrors ‚Äî `on_download`, `on_frame_attached`, `on_frame_detached`, `on_frame_navigated`, `on_page_close`, `on_page_load`. Currently these events fire only on `Page`; mirroring on `BrowserContext` matches the Playwright pattern of context-level cross-page observation and is a frequent request when writing multi-tab fixtures.
- [ ] `Browser::on("context")` ‚Äî fired when a new context is created on the browser; complements the existing `on("disconnected")` event and lets framework code observe context creation without going through `new_context()` return values.
- [ ] `Tracing::start_har(path, options)` / `Tracing::stop_har()` ‚Äî first-class HAR recording with `content`, `mode`, `url_filter` options, mirroring `BrowserContextOptions::record_har`. Returns a guard whose `Drop` finalizes the HAR (Rust analogue of the JS `await using` example).

**P2 ‚Äî Locator / Page / Assertion polish:**

- [ ] `Page::to_match_aria_snapshot()` page-level assertion ‚Äî equivalent to `expect(page.locator("body")).to_match_aria_snapshot(...)`. Adds the missing page receiver to the existing locator assertion.
