# v1.0 Feature Gap Analysis: playwright-rust vs playwright-python

**Last Updated:** 2026-05-16

**Baseline:** playwright-rust v0.12.0 vs playwright-python (Playwright v1.60+)

## Goal

Comprehensive API gap analysis comparing this repo's Rust bindings against the official Playwright Python bindings. Organized by priority to guide v1.0 development.

---

## Coverage Summary

Full Python API parity was achieved in v0.12.0. v0.13.0 (code complete
on `main`, awaiting tag cut) closes the remaining Playwright 1.59 gap
— `Page::screencast`, `Debugger`, `Page::pick_locator`, CDP event side
on `CDPSession`, `AriaSnapshotOptions`, `Tracing.start({live: true})`,
`Page::aria_snapshot`, `Locator::normalize`, `LaunchOptions::artifacts_dir`
— and adds two companion crates (`playwright-rs-macros` for the
compile-time-validated `locator!()` macro, `playwright-rs-trace` for
programmatic trace-file consumption). The bundled driver was bumped
from 1.59.1 → 1.60.0 on `main` on 2026-05-16; the bump is source-compatible
(none of the upstream-removed 1.60 APIs were exposed in this crate),
but the new 1.60 surface (HAR Tracing API, `Locator::drop`, expanded
BrowserContext lifecycle events, etc.) is tracked separately under
"v0.14.0 — Playwright 1.60 Completeness" below for incremental pre-v1.0
rollout. With v0.13.0 staged, playwright-rust matches the Playwright
1.59 surface and is slightly ahead of playwright-python.

v1.0.0 follows after multi-month dogfooding with real-world projects —
API polish, ergonomics refinements, and performance tuning rather than
new surface area.

### Errata

- This document previously listed `to_have_attribute`, `to_have_class`,
  `to_have_css`, and `to_have_count` under "Well-Covered Areas" since
  v0.9.0 — they were unimplemented. Reported in

## P0 — Critical for v1.0

### Locator: `get_by_*` Methods

These are *the* modern Playwright locator pattern. Without them, users cannot write idiomatic Playwright tests.

- [x] `get_by_role(role)` — Locate by ARIA role (most important)
- [x] `get_by_text(text)` — Locate by text content
- [x] `get_by_label(text)` — Locate by associated label
- [x] `get_by_placeholder(text)` — Locate by placeholder text
- [x] `get_by_test_id(test_id)` — Locate by test ID attribute
- [x] `get_by_alt_text(text)` — Locate by alt text
- [x] `get_by_title(text)` — Locate by title attribute

These must also be available on **Page** and **Frame** (they delegate to the main frame's locator).

### Locator: Filtering & Composition

- [x] `filter()` — Narrow locator with additional conditions (has_text, has, etc.)
- [x] `and_(locator)` — Match both this AND another locator
- [x] `or_(locator)` — Match either this OR another locator
- [x] `all()` — Return array of locators for each match

- [x] `all_inner_texts()` / `all_text_contents()` — Bulk text retrieval

### v0.14.0 — "Playwright 1.60 Completeness"

🆕 **Planned.** Tracks the new API surface introduced upstream in
Playwright 1.60 (released 2026-05-11). Driver itself bumped on
`main` 2026-05-16 — these items add Rust bindings for the new
methods/options. Priority labels match the P0/P1/P2/P3 buckets above.

**P1 — High value:**

- [ ] `Locator::drop(options)` — simulate external drag-and-drop of files or clipboard data onto an element. Dispatches `dragenter`/`dragover`/`drop` with a synthetic `DataTransfer`. Distinct from existing `drag_to(target)` (intra-page drag); this is the canonical primitive for upload-zone tests. Options: `files: Vec<FilePayload>` and/or `data: HashMap<String, String>` (MIME-typed string payloads).
- [ ] `BrowserContext` lifecycle event mirrors — `on_download`, `on_frame_attached`, `on_frame_detached`, `on_frame_navigated`, `on_page_close`, `on_page_load`. Currently these events fire only on `Page`; mirroring on `BrowserContext` matches the Playwright pattern of context-level cross-page observation and is a frequent request when writing multi-tab fixtures.
- [ ] `Browser::on("context")` — fired when a new context is created on the browser; complements the existing `on("disconnected")` event and lets framework code observe context creation without going through `new_context()` return values.
- [ ] `Tracing::start_har(path, options)` / `Tracing::stop_har()` — first-class HAR recording with `content`, `mode`, `url_filter` options, mirroring `BrowserContextOptions::record_har`. Returns a guard whose `Drop` finalizes the HAR (Rust analogue of the JS `await using` example).

**P2 — Locator / Page / Assertion polish:**

- [ ] `Page::to_match_aria_snapshot()` page-level assertion — equivalent to `expect(page.locator("body")).to_match_aria_snapshot(...)`. Adds the missing page receiver to the existing locator assertion.
- [ ] `boxes` option on `Locator::aria_snapshot()` / `Page::aria_snapshot()` — appends each element's bounding box as `[box=x,y,width,height]`. Useful for AI/LLM consumers building visual reasoning over ARIA trees.
- [ ] `description` option on `get_by_role()` (Page / Frame / Locator / FrameLocator) — match the WAI-ARIA accessible description in addition to name/role.
- [ ] `pseudo` option on `to_have_css()` — read computed styles from `::before` / `::after` pseudo-elements.
- [ ] `WebSocketRoute::protocols()` — returns the subprotocols requested by the page when handling a WebSocket route.
- [ ] `no_defaults` option on `BrowserType::connect_over_cdp()` — disables Playwright's default overrides (download behavior, focus/media emulation) when attaching to a user's running browser. Important for "attach without disturbing state" workflows.
- [ ] `WebError::location()` — already-existing mirror on `ConsoleMessage::location()`; adds page+filename+line+column to uncaught JS exceptions reported via `on("weberror")`.

**P3 — Minor polish:**

- [ ] `style` option on `Locator::highlight()` — extra inline CSS applied to the debug highlight overlay.
- [ ] `Page::hide_highlight()` — clears all highlights at once (companion to per-locator highlight).

**Deprecation handling (no public API break yet):**

- [ ] Migrate `ConsoleMessage::location()` callers from `lineNumber`/`columnNumber` to `line`/`column`. The 1.60 driver still emits both keys; the legacy ones are deprecated and may disappear in a future minor. Internal-only change.

**Out of scope (Playwright Test runner only, no language-binding equivalent):**

- `test.abort()` — `@playwright/test` test runner API
- `{testFileBaseName}` token in `snapshotPathTemplate` — Test runner
- `testInfoError.errorContext` / `reporter.onError(workerInfo)` — Test runner / reporters
